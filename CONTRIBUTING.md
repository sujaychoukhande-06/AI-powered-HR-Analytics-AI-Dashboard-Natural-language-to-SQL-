# Contributing to AI-Powered HR Analytics Dashboard

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## 🤝 How to Contribute

### Reporting Issues

1. Check if the issue already exists
2. Provide a clear description of the problem
3. Include:
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Your environment (Python version, OS, etc.)

### Submitting Changes

1. **Fork the repository**
   ```bash
   Click "Fork" on GitHub
   ```

2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/AI-powered-HR-Analytics-AI-Dashboard-Natural-language-to-SQL-.git
   cd AI-powered-HR-Analytics-AI-Dashboard-Natural-language-to-SQL-
   ```

3. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make your changes**
   - Keep changes focused and clear
   - Test your changes thoroughly

5. **Commit with clear messages**
   ```bash
   git commit -am 'Add new feature'
   ```

6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Create a Pull Request**
   - Provide a clear description
   - Reference any related issues

## 💻 Development Guidelines

### Code Style

- Follow PEP 8 conventions
- Use clear, descriptive variable names
- Add comments for complex logic
- Keep functions focused and small

### Testing

- Test your changes thoroughly before submitting
- Ensure the dashboard works with different databases
- Test with various query types

### Documentation

- Update README.md if adding features
- Document any new functions
- Add examples for new functionality

## 📝 Commit Message Guidelines

Use clear commit messages:

```
feat: Add feature description
fix: Fix bug description
docs: Update documentation
refactor: Refactor code section
```

## 🙏 Code of Conduct

- Be respectful and professional
- Provide constructive feedback
- Collaborate positively with others

Thank you for contributing! 🎉
